# Tiny Market 요구사항 분석

## 1. 프로젝트 목적
사용자가 중고 상품을 등록하고 다른 사용자와 거래할 수 있는 플랫폼을 개발한다.

## 2. 사용자 역할
- 일반 사용자
- 관리자

## 3. 핵심 기능
### 회원
- 회원가입
- 로그인 및 로그아웃
- 본인 정보 조회

### 상품
- 상품 등록
- 상품 목록 조회
- 상품 상세 조회
- 상품 검색
- 본인이 등록한 상품 수정 및 삭제

### 채팅
- 구매자와 판매자의 채팅방 생성
- 메시지 전송 및 조회

### 차단
- 악성 사용자 차단
- 부적절한 상품 차단

### 송금
- 사용자 간 가상 포인트 송금
- 송금 내역 조회

### 관리자
- 사용자 관리
- 상품 관리
- 차단 상태 관리

## 4. 보안 요구사항
- 비밀번호를 평문으로 저장하지 않는다.
- 모든 입력값을 서버에서 검증한다.
- 로그인 여부와 사용자 권한을 서버에서 확인한다.
- 본인의 상품만 수정하거나 삭제할 수 있다.
- 송금 금액과 잔액을 서버에서 검증한다.
- 관리자 페이지는 관리자만 접근할 수 있다.
- 비밀키와 데이터베이스 주소를 GitHub에 올리지 않는다.

## 5. 개발 범위
- 실제 현금 대신 가상 포인트를 사용한다.
- 이미지 업로드는 초기에는 이미지 URL 방식으로 구현한다.
- 실시간 채팅은 기본 기능 완성 후 확장한다.
